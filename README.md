# Optimization Algorithms Collection
Коллекция алгоритмов оптимизации, реализованных в рамках учебного проекта. Каждый алгоритм представлен в виде отдельного класса, наследуемого от `BaseAlgorithm`, и зарегистрирован в фабрике для использования в веб-интерфейсе.

## 1. Градиентный спуск (Gradient Descent)
Классический метод оптимизации первого порядка с постоянным шагом.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `lr` | Шаг обучения | 0.1 |
| `epsilon1` | Критерий остановки по норме градиента | 0.1 |
| `epsilon2` | Критерий остановки по изменению точки и значения | 0.15 |
| `max_iter` | Максимальное число итераций | 10 |
| `x0` | Начальная координата x₁ | 0.5 |
| `y0` | Начальная координата x₂ | 1.0 |
Условия остановки
1. Норма градиента меньше $\varepsilon_1$
2. Дважды подряд $|\triangle x|<\varepsilon_2,|\triangle f|<\varepsilon_2$
3. Достигнуто `max_iter`

## 2. Пчелиный алгоритм (Bee Algorithm)
Алгоритм, имитирующий поиск пищи пчелиной колонией. Сочетает глобальный поиск (разведчики) и локальный (рабочие пчёлы).

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `scouts` | Число пчёл-разведчиков | 16 |
| `elite_sites` | Число элитных участков | 2 |
| `best_sites` | Число перспективных участков | 3 |
| `elite_bees` | Число пчёл на элитном участке | 7 |
| `best_bees` | Число пчёл на перспективном участке | 4 |
| `radius` | Радиус локального поиска | 0.2 |
| `max_iter` | Максимальное число итераций | 500 |
| `stagnation` | Число итераций без улучшения для остановки | 20 |
| `x_min` / `x_max` | Область поиска | -5.0 / 5.0 |

## 3. Генетический алгоритм
Эволюционный алгоритм с турнирной селекцией, равномерным скрещиванием и элитизмом.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `pop_size` | Размер популяции | 100 |
| `generations` | Число поколений | 200 |
| `mutation` | Вероятность мутации | 0.1 |
| `elite_frac` | Доля элитной популяции | 0.1 |

***Особенности:***
- Турнирная селекция (k=3)
- Равномерное скрещивание (uniform crossover)
- Гауссовская мутация
- Элитизм сохраняет лучшие особи

## 4. Алгоритм роя частиц (PSO)
Метаэвристический алгоритм, вдохновлённый социальным поведением стай.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `n_particles` | Число частиц в рое | 30 |
| `iterations` | Число итераций | 100 |
| `w` | Коэффициент инерции (скорости) | 0.1 |
| `c1` | Локальный коэффициент (личный опыт) | 2.0 |
| `c2` | Глобальный коэффициент (социальный опыт) | 5.0 |
| `x_min` / `x_max` | Область поиска | -5.0 / 5.0 |

***Формула обновления скорости:*** $v=\chi(v+c_1r_1(p_{best}-x)+c_2r_2(g_{best}-x))$, где $\chi$ $-$ коэффициент сходимости, вычисляемый автоматически.

## 5. Бактериальная оптимизация (BFO)
Алгоритм, имитирующий поведение бактерий при поиске пищи.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `n_bacteria` | Число бактерий (чётное) | 20 |
| `t_chemo` | Шаги хемотаксиса | 30 |
| `t_repro` | Шаги репродукции | 4 |
| `t_elim` | Шаги ликвидации и рассеивания | 2 |
| `lambda_` | Величина шага хемотаксиса | 0.1 |
| `p_elim` | Вероятность ликвидации бактерии | 0.25 |
| `n_elim` | Число ликвидируемых бактерий | 2 |
| `x_min` / `x_max` | Область поиска | -5.0 / 5.0 |

***Фазы алгоритма:***
1. **Хемотаксис** $-$ движение бактерий в градиентном направлении
2. **Репродукция** $-$ лучшие бактерии размножаются
3. **Ликвидация и рассеивание** $-$ случайная замена части популяции

## 6. Искусственная иммунная сеть
Алгоритм, основанный на принципах клональной селекции иммунной системы.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `pop_size` | Размер популяции антител | 50 |
| `n_b` | Лучшие антитела для клонирования | 10 |
| `n_c` | Число клонов на антитело | 10 |
| `n_d` | Лучшие клоны после мутации | 5 |
| `alpha` | Коэффициент мутации | 0.5 |
| `max_iter` | Максимальное число итераций | 200 |
| `x_min` / `x_max` | Область поиска | -5.0 / 5.0 |

***Этапы:***
1. Отбор лучших антител (`n_b`)
2. Клонирование (`n_c` клонов на антитело)
3. Гипермутация клонов
4. Отбор лучших клонов (`n_d`)
5. Обновление популяции

## 7. Квадратичное программирование
Метод решения задач квадратичного программирования с использованием `scipy.optimize.minimize` (метод SLSQP).

***Особенности:***
- Автоматически извлекает параметры задачи из функции
- Поддерживает линейные ограничения-неравенства
- Использует аналитический градиент
